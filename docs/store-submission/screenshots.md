# Screenshot And Capture Plan

> Capture plan for store assets. Actual screenshots, trailers, and Meta dashboard uploads are
> external submission steps and are not performed by repository automation.

## Required Capture Set

- Boot/main interaction scene with the branded startup overlay dismissed.
- Creative building: active block placement with authored block textures visible.
- Survival-lite terrain: caves/resources visible from normal player height.
- Multiplayer LAN: two players connected with the session status visible.
- Comfort/UI: comfort menu or block menu readable in headset.

## Capture Rules

- Capture on Quest 3 for the first store candidate; repeat on Quest 3S when available.
- Use the signed release APK from a `main` tag for final assets.
- Keep controller rays, debug overlays, editor gizmos, and development-only UI out of final
  store screenshots.
- Do not include third-party brands, protected character identities, or non-original art.

## Local Evidence

Record local validation evidence separately from final store assets:

- Quest smoke screenshots or recordings may be attached to GitHub issues/PRs.
- Performance evidence belongs under `docs/testing/performance/`.
- Final dashboard asset dimensions must be confirmed against Meta's current submission form
  immediately before upload.
