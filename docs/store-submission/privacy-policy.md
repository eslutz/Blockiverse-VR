# Blockiverse VR Privacy Policy Draft

This is a repository draft for review. It is not yet the public privacy policy URL required for a Meta Horizon Store submission.

## Overview

Blockiverse VR is an immersive voxel sandbox game for Meta Quest 3 and Meta Quest 3S. The first store candidate is designed around local gameplay and LAN co-op. It does not include public matchmaking, public world discovery, in-app voice chat, in-app text chat, ads, or a Blockiverse account system.

## Data Processed By The App

Blockiverse VR may process the following data to run the game:

- Local save data, including world state, block edits, inventory state, crafting state, health state, and local settings.
- Local runtime diagnostics, including Unity player logs, warnings, errors, performance observations, and validation notes captured during development or testing.
- LAN session data, including local network host/client session state and gameplay messages required to synchronize nearby players.
- Meta Horizon Avatar and platform profile data when Meta platform avatar features are active in a build.
- Device and build information needed by Unity, Meta Quest runtime services, and the operating system to launch and run the app.

## How Data Is Used

Data is used to:

- Load, save, and restore the player's game world.
- Run creative, survival-lite, crafting, inventory, and LAN co-op gameplay.
- Display the player's Meta Horizon avatar where platform avatar integration is active.
- Diagnose local runtime issues, crashes, performance problems, and store-candidate validation failures.
- Meet platform review, support, and compliance obligations.

## Data Not Collected Or Hosted By Blockiverse VR In The First Store Candidate

The first store candidate does not include:

- In-app voice recording or voice transport.
- In-app text chat or public user-generated text.
- Public matchmaking or a public community world browser.
- Cloud-hosted persistent worlds.
- Ads or advertising identifiers.
- A custom Blockiverse account, custom avatar profile, or custom player identity system.
- Sale, rental, or licensing of user data.

Players may choose to use Meta Quest party chat outside the app. That communication is provided by Meta platform services, not by Blockiverse VR.

## Storage And Retention

Local save data remains on the player's device unless the player exports, deletes, backs up, or otherwise transfers it using platform or device tools. Local development logs and validation captures should not be committed to the repository; when needed for support or review, relevant excerpts may be attached to a GitHub issue, pull request, or private support channel.

## Sharing

Blockiverse VR does not share first-candidate gameplay data with third-party analytics, advertising, or marketing services. Meta platform services, Unity runtime services, the device operating system, and the Meta Horizon Store may process data according to their own terms and policies when the app runs on Meta Quest hardware.

## Deletion Requests

Players can remove local game data by uninstalling the app or clearing app data through device settings. A public support path must be published before external submission so players can request deletion or ask privacy questions for any data Eric Slutz controls outside the device.

## Policy Publication Status

Before real store submission, this draft must be reviewed, approved, and published at a publicly accessible HTTPS URL. The published URL must be entered in the Meta Developer Dashboard privacy policy field and kept current.
