# Changelog

All notable changes to Blockiverse VR will be documented here.

The format is based on Keep a Changelog, and releases use tags cut from `main`.

## Unreleased

- Completed the deep-review follow-up pass over the 131 remaining unverified medium/low observations: 121 were rejected with documented evidence (already remediated, deliberate design, no concrete failure path, false positive, or fix would break determinism/save compatibility) and the 10 valid ones were fixed — torchbud lights placed beyond the 24-light cap now queue and ignite when a slot frees instead of staying dark for the session, the crafting panel keeps station-gated recipes unlocked across inventory rebinds, station panel labels show real item display names, `CraftingRecipe` rejects negative craft times, sapling scanning uses the flat-array sweep, and two worldgen ordering/unit traps are documented in place. Added regression tests for surface resource node placement, world-clock restore phase math, and crop growth interval anchoring (EditMode suite grows to 379).
- Expanded `CLAUDE.md` from a one-line pointer into the repository orientation doc (developer commands including single-test invocation, assembly layering map, cross-cutting invariants, runtime flow, save format) and pointed `AGENTS.md` at it for command/architecture orientation; workflow policy stays in `AGENTS.md`.
- Added two future features to the execution plan: hand tracking and hand motion control (V2 target — hands as an optional input mode alongside controllers) and mixed reality room play (V3 target — passthrough mode anchoring a room-scaled voxel volume in the player's real space with persistent spatially anchored block placement).
- Remediated the full PR #310 deep-review finding set (18 confirmed high-severity issues plus 20 suggestions): closed save-data loss windows (atomic-replace fallback now restores the original on failure, multiplayer host saves no longer reset difficulty/world preset, survival-restore inventory overflow is surfaced instead of silently dropped, legacy weather restore keeps deterministic state); hardened the multiplayer trust boundary (host re-stamps avatar stream sender identity, station snapshots guard malformed item ids, snapshot generation recovers across component disable/enable, chunk-delta sequence ids stay monotonic across clears); cut Quest 3 runtime cost (bulk avatar payload serialization instead of per-byte calls, chunk meshes no longer cast/receive real-time shadows, pooled chunk-mesh build lists, pre-aggregated recipe ingredients, cached head transform and UI refresh gating, flat-array leaf-decay seeding); fixed the death-screen menu escape that left the player undead; made world-name sanitization platform-independent (Android returns no invalid-filename chars) and bounded save-slot allocation; validated PlayerPrefs enum loads; decorrelated cave radii from cave centers (bit-shift aliasing); and made the rig-spawn helper public to fix a compile break in the session controller that shipped in PR #310 (CI does not compile Unity code; `scripts/unity/run-tests.sh` is the compile gate). Also added the four missing item inventory icons (worldroot, deepmantle, snowpack, frostglass) required by art-asset validation. Added regression tests for difficulty/world-preset save round-trips, survival inventory snapshot validation, loot-table and lighting edge cases, and made the multiplayer PlayMode suite safe under parallel execution.
- Added the generated music bed and its playback logic: four original synthesized tracks (menu, day, night, cave — pad chords with seeded pentatonic melodies from the reusable audio generator, imported as streaming clips), a rig-level music controller that resolves the playback context (menu before a world exists; cave via the sky-access map; day/night via the world clock's shared daylight window, now centralized on `WorldTimeClock.IsDay`), plays one track at a time with fade-in/out envelopes and long randomized silence gaps between tracks, ducks the current track on context changes, and follows a new Music volume category end to end (feedback settings × master × mute, an Audio & Feedback panel slider, and settings persistence).
- Added the §5.4 fluid flow simulation: freshwater/brine/emberflow sources now spread flowing cells (per-family distance and tick cadence — 8/5, 6/6, 4/12) that fall first (a drop restarts the horizontal budget), retract when their source support is lost, and merge per fluid family on the chunk fluid mesh; emberflow glows, deals contact heat damage through the shared hazard table, deterministically burns adjacent wood/foliage away, and is quenched into black basalt by freshwater contact; deep caves (below y=18) roll 10% emberflow pools at generation; the simulation is a pure function of the synced world, the shared world clock's absolute tick, and the world seed (the crop-growth lockstep pattern — flow costs zero network traffic, and late joiners/loads rebuild flow budgets from the world via a multi-source BFS), and the world manager now repaints dirty chunks every tick so simulation mutations are visible without waiting for a player edit.
- Validated the slice with a full bootstrapper regen and the EditMode + PlayMode suites (403 + 63 green): guarded the pooled chunk-mesh collider against empty (0-vertex) meshes, dropped a redundant persistent `MenuPressed` listener the menu controller already subscribes at runtime, derived inventory icons for block-mapped items that lacked one (worldroot/deepmantle/snowpack/frostglass) and the freshwater/brine block source tiles from the bootstrapper, and made the cave shrine the reliable all-biome loot source (cleared interior, crate always placed, smaller spawn keep-out) so structure loot stays present now that lakes skip flooded columns.
- Added locomotion and loot feedback: stride-timed footstep cues and a landing thump driven by the rig's character controller during glide movement, and container-open/pickup cues when a broken structure crate grants its loot.
- Added two rare structures: a cave shrine (cutstone base, corner pillars, lumen lamp, loot crate) set into the first cave pocket beneath its column with a surface fallback, and a weathered plank bridge crossing — both deterministic, low-weight catalog entries.
- Pooled chunk meshes in the voxel renderer: each chunk (and its fluid child) keeps one persistent Mesh that is cleared and refilled per rebuild instead of allocating and destroying a new Mesh every edit, removing the deferred collider-mesh bookkeeping (the throttled collider keeps serving its cooked snapshot until its rebake recooks the same instance).
- Added reconnect identity for LAN co-op: each player carries a persistent GUID introduced to the host on connect; the host stashes a departing player's inventory under it and hands it back when the same player rejoins the session.
- Made survival mining a timed hold (§7.3): pressing break starts a hold on the aimed block whose duration comes from the harvest rule's work time with the equipped tool, with tool-strike cues and chip VFX on a cadence while held; releasing or losing the aim cancels, blocked harvests (wrong tool, full inventory) submit immediately so the rejection feedback plays, and creative breaking stays instant.
- Added icon-based item UI: a rig-level item icon library populated by the bootstrapper from the committed item icon sprites (importers forced to Sprite), inventory slots now render the item icon plus a count with a text fallback for icon-less items, and crafting rows show their output item's icon.
- Completed the creative toolset: a Creative Tools screen on the pause menu with corner A/B region selection from the aimed-at block, fill/replace/delete/copy/paste through the restored region world-edit service (§12.1 volume caps, own undo/redo history, gated to offline creative worlds), tree/ruin spawners, a pick-block action, and environment controls (time-of-day and day-speed sliders, weather cycling through every preset); upgraded the left-grip blocks menu into a catalog browser (category cycle, paged 3×4 pick grid, system-keyboard name search) backed by the full creative catalog; and added a per-block redo stack alongside undo (standard branch-invalidation semantics, host-authoritative).
- Completed the remaining menu screens: the Settings hub now routes to Comfort, a new Audio & Feedback panel (master/effects/UI/weather volume and haptic-strength sliders plus mute/haptics/reduced-flash/reduced-particles toggles, persisted with the existing settings snapshot), and a read-only Controls reference (sharing one canonical mapping text with the first-launch popup, which was stale); added the §6.5 World Details screen (metadata, rename field, Play/Rename/Duplicate/Delete/Back) reached from a Details button on Load World, with save rename/duplicate/delete file operations, a delete confirmation modal, and a generic single-OK error dialog.
- Added host-only environment dynamics driven by the weather machine: thunderstorm lightning strikes that scorch what they hit (meadow turf chars to dry turf, leafmoss burns away) with spawn/player-proximity exclusion and a strike event for feedback layers, plus tundra snow that accumulates one snowpack layer on exposed columns during snowfall and melts under clear skies — all applied through the chunk-authority mutation channel so clients receive ordinary deltas.
- Rebuilt the `void_builder` world preset to the §11.3 spec (a 16×16 cutstone starting platform centered at the spawn column, nothing else generated) and wired it through the New World preset selector, world creation, save regeneration, and the multiplayer late-join snapshot.
- Completed world persistence beyond blocks/inventory and added autosave: saves now carry the player's position/heading and vitals (health/hunger/thirst/stamina), the full weather-machine position (state + ticks + RNG, restored exactly instead of restarting the weather timeline), a `simulation.json` with sapling growth progress, wild-regrowth markers, and pending berrybush regrowth, and a `stations.json` with kiln/forge slot contents and in-flight craft progress; single-player sessions and hosted multiplayer worlds autosave on the 5-minute save-service cadence, and stale simulation/station files are deleted when their state empties.
- Added static fluids per the §5.4 ruleset: freshwater/brine source blocks (non-solid, light-passable, rendered on a contact-excluded per-chunk fluid mesh so players wade through water that rays can still target), worldgen lakes/seas filling columns below sea level (brine in the dunes) with 35% flooded caves, fluid-aware carving/vegetation/structure guards, dry-land spawn search, drinking from freshwater sources by hand, the bucket chain (craft empty bucket → fill at a source → pour back, host-authoritative FillBucket/PourBucket commands), the water-flask chain (kiln-fired water flask → campfire clean water flask with the emptied bucket returned), brine boiling to brightsalt, crafting byproducts with all-or-nothing rollback, container returns on consume (§731), the §11.1 tilling water requirement (freshwater within reach or one clean water flask), and atlas tiles painted additively by the bootstrapper.
- Made hunger satisfiable (the survival food loop closes): grain bundles and berry clusters are consumables that restore hunger (berries also restore a little thirst), and drinking restores thirst from clean water flasks or directly from world freshwater on a short cooldown.
- Wired the single-player session flow end to end: the menu controller's actions now drive a world-session coordinator that creates worlds from the New World config (seed/size/preset/difficulty + starting-biome spawn search), saves/loads directory-schema saves under `Saves/`, continues the most recent save, and keeps the title menu's Continue/Load entries and Load World list current; added a LAN Multiplayer entry and panel; and put the full survival/network runtime stack (chunk authority, survival sync, vitals, persistence) into the Boot scene so survival mode and hosting/joining are reachable without a scene switch.
- Fixed several correctness and security findings: the offline player inventory is now a single shared instance (no more harvest/craft results landing in an invisible inventory); timed kiln/forge recipes can no longer be crafted instantly with no fuel; survival worlds reject the raw creative mutation channel from clients; client→host message handlers no longer throw on malformed payloads; corrupt saves fail controlled again (bounds-validated region deltas, validated survival snapshot, region-swap crash recovery); container loot is granted host-authoritatively to the breaking player; and the world clock no longer shifts time-of-day on load/late-join.
- Reduced VR runtime cost: removed a per-frame `ScriptableObject` leak and per-frame string lookups in the input rig, replaced the per-block-edit lighting-rebuild storm and full sky-column walks with an incremental per-column sky-light map, moved late-join world regeneration off the main thread, persisted comfort/feedback settings across launches, and bounded the chunk-delta log.
- Generated the previously missing resource nodes and terrain so the tier 2–7 tool/recipe ladder is obtainable (paletin/niter/sunmetal/umbralite/staropal veins, per-biome surface nodes, biome surfaces, and depth strata), and wired survival action + weather feedback (audio/VFX/haptics for harvest/place/craft, weather/ambience loops, thunder/lightning, campfire and torch cues, multiplayer join/leave).
